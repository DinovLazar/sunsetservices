## Open items / known mismatches with the Plan

- **Lighthouse mobile Performance gap (Phase 1.07).** Desktop EN/ES hit 98/97/96/100 on Performance / Accessibility / Best Practices / SEO — all ≥ 95 ✓. Mobile EN/ES hit **86** / 97 / 96 / 100 — Accessibility, Best Practices, SEO all ≥ 95 ✓ but **Performance lands ~9 points below the 95 target**. Root cause is structural: simulated 4× CPU mobile throttle + the page's locked design (7 sections, 20 images, motion-react entrance choreography on six of seven sections) yields a 1.2 s LCP element render delay even after the hero image arrives at ~130 ms. Mitigations attempted: pinned hero `height` (vs `min-height`) so its bounding rect doesn't drift during font swap, regenerated hero placeholder with high-entropy noise (so Lighthouse picks the photo as LCP, not the H1), `content-visibility: auto` on every below-hero section, `loading="lazy"` audit, removed `text-wrap: balance` on the hero H1. Each shaved a few points; ceiling appears to be ~86. Realistic paths to 95+ are (1) refactor Phase 1.04 motion primitives to CSS-only IntersectionObserver entrances (drops motion-react from below-hero hydration cost), (2) Phase 2.04 real photographs (likely smaller and more compressible than entropy-tuned placeholders), (3) consider `next/dynamic` of below-hero sections with `ssr: true`. None of these are in Phase 1.07 scope.

- Real env vars NOT populated — `.env.local.example` documents the future variables; `.env.local` is gitignored and empty.
- `Sunset-Services-Project-Instructions.md` lives in the Claude project knowledge base; a copy is now in `src/_project-state/` for reference.

This is the canonical `next-intl` pattern; the handover's example file tree showed both, but only one root layout is permitted by Next.js. Documented in `00_stack-and-config.md`.
- **Featured-card discipline (D2, ratified):** any future page-level handover that sets `featuredCard: true` on a section must NOT also include an amber CTA on that same page. Carry forward from 1.06+. Phase 1.07 honors this — no `.card-featured` on the homepage.

The locked Phase 1.03 components in `globals.css` use single-hyphen BEM (`.btn-ghost`, `.card-photo`, `.btn-amber`). Per the handover header note ("if the handover conflicts with Phase 1.03 tokens, the earlier phase wins"), Phase 1.07 implemented the on-dark modifier as `.btn-ghost.btn-on-dark`. All section components use single-hyphen class names throughout to stay consistent with Phase 1.03.

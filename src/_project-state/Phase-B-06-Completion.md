## Headline

The new `scripts/validate-a11y.mjs` harness (wired as `npm run validate:a11y`) drives Playwright headless Chromium across an 18-URL representative set: 15 EN URLs (one per route family + the four legal/auth routes) + 3 ES parity URLs (home, service detail, wizard). Per URL it runs `@axe-core/playwright` with the six WCAG tags `wcag2a` / `wcag2aa` / `wcag21a` / `wcag21aa` / `wcag22a` / `wcag22aa`, plus a Lighthouse a11y category run via `chrome-launcher`, plus programmatic checks for WCAG 2.2 net-new SCs 2.4.11 (Focus Not Obscured) and 2.5.8 (Target Size Minimum). It exits 0 only when every URL has zero AA violations AND every Lighthouse a11y score ≥ 95.

Initial sweep surfaced one cross-cutting AA failure and three repeating ambiguity findings:
- **`color-contrast`** — white text on `--color-sunset-green-500` (#4D8A3F) gives 4.18:1, below the 4.5:1 AA threshold. Fired on every CTA button (`.btn-primary` base + the `.link:hover` color swap) across all 18 URLs (23 total nodes).
- **`aria-prohibited-attr`** — two placeholder `<div aria-label="Unilock...">` elements that lack a role (one in the footer, one on the homepage credentials strip).
- **`aria-valid-attr-value`** — the two navbar mega-panel buttons reference `aria-controls="services-mega-panel"` / `"resources-mega-panel"`, but those panels mount lazily via `AnimatePresence`; the IDs don't exist in the DOM while closed.
- **`nested-interactive`** — the service-areas SVG map declares `role="img"` but contains real `<Link>` pins (focusable children); SC 4.1.2 prohibits focusable descendants of `img`-role elements.
- **WCAG SC 2.5.8 target-size** — the navbar desktop phone link (`<PhoneLink>`'s `xl:` variant) renders 164×16 because the desktop tier dropped `min-h-[44px]` for tight navbar spacing.

The fix-pass introduced a new `--color-sunset-green-600 = #3F7335` token (5.2:1 white-on-green AA contrast) used by `.btn-primary` base + `.link:hover` and three Tailwind callsites; added `role="img"` to the two Unilock placeholder divs; made `aria-controls` conditional on the open state for both the navbar mega-panel triggers AND (defensively) the chat bubble; refactored the service-areas SVG into a `<nav aria-label>` wrapper with the SVG's `role="img"` dropped; and bumped the desktop PhoneLink to `xl:min-h-[24px] xl:min-w-[24px]` to satisfy SC 2.5.8. A `role="region"` + locale-aware `aria-label` was added to the Termly embed wrapper for landmark semantics.

After the fix-pass, the harness exits 0 with totals: **0 axe AA violations, 0 best-practice findings, 10 incomplete (color-contrast on photo+gradient backgrounds — visually verified, see §4), 0 WCAG 2.2 SC 2.4.11, 0 WCAG 2.2 SC 2.5.8, all 18 URLs Lighthouse a11y = 100/100, `prefers-reduced-motion` matchMedia plumbing verified.**

Both pre-existing regression harnesses still PASS: `npm run validate:schema` → 0/0 across 22 URLs; `npm run validate:seo` → 0/0 across 118 URLs + sitemap + robots.

## WCAG 2.2 net-new AA SC verification

Per D7 of the plan, the five WCAG 2.2 SCs added at AA were explicitly verified:

### 2.4.11 Focus Not Obscured (Minimum) — Programmatic

Per-URL Playwright walk: focuses every visible interactive (cap 200/page), captures its bounding box, then per-element checks every `position: fixed` / `position: sticky` overlay's intersection with the focused box. If any overlay covers > 50% of the focused element's area, it fires a finding.

**Result: 0 findings across all 18 URLs.** The navbar (`position: sticky`), chat bubble (`position: fixed`), and cookie banner (`position: fixed`) do not obscure any focused element by more than 50% on any page. The wizard sticky-Next bar (only visible on `/request-quote`) also clears the threshold — the wizard's main content has sufficient bottom-padding so focused fields stay above the sticky bar.

### 2.5.7 Dragging Movements — Code review

Audited every surface with potential drag interactions:
- **Project gallery lightbox** (`ProjectGallery.tsx`) — no drag-to-pan. Uses keyboard arrow keys + click on Prev/Next/Close buttons. ✅
- **Mobile chat bottom-sheet** (`ChatPanel.tsx`) — `<dialog>.showModal()` pattern; closes via Esc, click-outside, or the menu's Close button (kebab → Close). No drag-to-dismiss. ✅
- **Wizard steps** — no drag interactions; selections are tap/click + keyboard. ✅
- **Filter chip strips** (`projects/FilterChipStrip.tsx`, `content/FilterChipStrip.client.tsx`) — uses CSS `scroll-snap-type: x mandatory` on a horizontal scroll container. The scroll is a native browser horizontal scroll, which works via wheel, keyboard (←/→), or touch — touch-scroll is the same gesture as a drag, but the user can also tap the chips directly to select. The chips themselves are real buttons/links; no drag is required to operate them. ✅

**Result: All drag interactions have a non-drag alternative.**

### 2.5.8 Target Size (Minimum) — Programmatic

Per-URL DOM walk: collects every visible interactive's bounding rect. Sub-24px hits are flagged unless either (a) the element is inline-in-text (link inside `<p>`, `<li>`, `<span>`, etc. with a text sibling) OR (b) the SC spacing exception applies (no other interactive's center is within 24px center-to-center distance). The spacing exception implementation matches the SC's "24 CSS pixel diameter circle centered on the bounding box of each target does not intersect any other target's circle" — center-to-center separation ≥ 24px ⇒ exempt.

**Initial sweep:** 1 finding — the navbar's desktop PhoneLink (164×16, no spacing exception because the adjacent "Get a Quote" CTA is within 24px). **Fixed:** bumped `xl:min-h-[24px] xl:min-w-[24px]` so the desktop tier of `PhoneLink` clears the 24px threshold even with the tight navbar `min-h-[44px]` exemption.

**Result after fix: 0 findings across all 18 URLs.**

### 3.2.6 Consistent Help — Code review

The three "help mechanisms" referenced in the SC are:
- **Privacy link** — in `FooterLegal.tsx`, rendered consistently in the footer's bottom strip on every page.
- **"Get a Quote" CTA** — `NavbarGetQuoteCTA.tsx`, rendered consistently in the navbar's right side on every page (hidden only on `/request-quote/` itself; the SC doesn't require the mechanism to appear on the page that IS the destination).
- **Phone number** — `PhoneLink.tsx` in both the desktop navbar (right side) and the footer brand block. Position is consistent across all pages.

**Result: order + position consistent across pages. SC met.**

### 3.3.7 Redundant Entry — Code review

Audited `src/data/wizard.ts` (`WIZARD_STEP_1_FIELDS` through `WIZARD_STEP_4_FIELDS`) + each Step component:
- Step 1 collects: audience (one of residential / commercial / hardscape).
- Step 2 collects: service (one of the 16 services in the chosen audience).
- Step 3 (audience-conditional): propertySize, bedrooms, projectType, timeline, budget, notes (residential); numProperties, numBuildings, contract, frequency, timeline, notes (commercial); spaceType, dimensions, surface, features, budget, timeline, notes (hardscape).
- Step 4 collects: firstName, lastName, email, phone, street, unit, city, state, zip, bestTime, contactMethod.

Step 4 introduces fields NOT previously collected — contact identity + delivery address + scheduling preferences. There is no overlap with Steps 1–3. **Result: SC met.**
